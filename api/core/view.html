视图对象

定义与创建

var View = app.view.define();
var view = new View(options);
view.render();

包含Backbone 中已有的属性或方法，常使用的有：

  • \$el
  • template
  • \$()
  • trigger
  • on
  • listenTo
  • stopListening 等

配置项

autoAction

  • Type: Boolean
  • Default: false

自动绑定Action事件

举例说明，当在模板中使用如下写法时：

<button data-action="add">添加</button>

如果该属性为 true，将自动查找该视图的 addHandler 方法作为该按钮点击事件的处理函数

autoRender

  • Type: Boolean
  • Default: true

自动渲染. 视图一初始化就进行渲染

autoResize

  • Type: Boolean
  • Default: false

自适应窗口变化. 该属性设为true后，当窗口大小变化时，会自动调用resize方法，因此需要重写该方法

autoST

  • Type: Boolean
  • Default: false

自动设置触发器. 该属性为true后，会广播 setTriggers 消息，可将该视图的工具条注入到其他插件，需要 额外设置 toolbar 项，指定该视图的注入到的插件名称

switchable

  • Type: Array
  • Default: []

可切换的视图. 填写可切换的视图的名称，可实现多个子视图的切换显示效果

toolbar

  • Type: String
  • Default: null

配合 autoST 使用

sharedModel

  • Type: Object
  • Default: null

传入一个视图模型对象，用来初始化该视图的视图模型

sharedModelProp

  • Type: Array
  • Default: null

sharedModel 的字段筛选

sandbox

  • Type: Object
  • Default: null

Sandbox 对象的实例，当作为插件主视图时，不需要配置，当作为子视图时，需要从主视图传递该对象

langClass

  • Type: String
  • Default: null

从 app.lang 读取配置的语言字典

bindEmptyModel

  • Type: Boolean
  • Default: false

是否绑定空对象

windowOptions

  • Type: 'Object'

设置当该视图呈现在窗口中时，该窗口的一些属性

windowOptions: {
    width: 500,
    height: 400,
    title: '窗口'
}

可自定义的方法

aspect

AOP 某些方法

aspect: function(){
    this.after('initAttr', function(){
        this.param = { test: 'A' }  
    });

    this.before ...
}

subscribe

监听其他插件的消息

subscribe: function(){

    this.sub('setTriggers', function(){
        alert('I received this message');
    })

    this.sub ...
}

listen

监听子视图的事件

listen: function(){
    this.listenTo('rendered', function(){
        // 处理代码
    });

    this.listenTo ...

    this.listenToDelay('edit', 'saved', function(){

    })
}

enhance

在 render 过程中，需要自定义的一些行为，通常放置一些不能被绑定初始化的控件初始化代码

enhance: function(){
    this.$('.chart').chart({
        type: 'pie',
        data: ['0.3', '0.2']
    })
}

init

视图的自定义初始化代码

initAttr

设置视图的自定义属性

initAttr: function(){
    this.message = 'hello';
    this.baseModel = {
        data: {
            name: 'veronica'
        }
    }
}

resize

当开启 autoResize 后，窗口大小变化时，该方法会被调用，该方法应编写窗口大小变化时，该视图对应的处理逻辑

delegateModelEvents

视图模型的事件处理

delegateModelEvents: function(){
    var viewModel = this.model();
    viewModel.bind('change', function(){
        // 处理代码
    });
    viewModel.bind('change.xxx', function(){ });
}

类属性

binds

  • Type: Array
  • Default: ['this']

设置哪些方法应该将该方法的执行上下文绑定到该视图的 this

template

  • Type: Function or String
  • Default: null

设置该视图的模板，一般情况下直接传入字符串即可

{ 
    template: tpl
}

defaults

设置该视图的默认参数

views

  • Type: Function or Object

配置该视图的子视图

实例属性

options

视图的配置项

options.sandbox

  • this.options.sandbox

当视图在插件中运行时，通过 sandbox 访问框架方法和进行插件间通信

可调用的方法

instance

  • this.instance(selector)

获取某个元素下的kendo对象实例

window

获取/设置子窗口

view

获取/设置子视图

model

设置视图模型

show

显示该视图

hide

隐藏该视图

destroy

销毁该视图

render

渲染该视图

after

注入某个方法，在某个方法后运行

var dialog = new Dialog();

dialog.after('show', function(returned, a, b) {
    console.log(returned); // show 的返回值
    console.log(a);
    console.log(b);
});

dialog.show(1, 2); // ==> undefined, 1, 2

before

注入某个方法，在某个方法前运行

replaceTpl

替换模板

attr

获取或设置视图的属性
